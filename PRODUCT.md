# Product

## Register

product

## Users

Developers who already use Pi and want to run, supervise, and resume one or more Pi-backed coding sessions across local projects from a native macOS workbench.

## Product Purpose

PiLot brings projects, concurrent sessions, structured agent activity, approvals, and read-only change inspection into one focused Mac workspace while preserving Pi as the engine and existing development tools as the place for editing and Git delivery. Success means users can direct work, notice sessions needing attention, understand what happened, inspect changes, and hand off without navigating terminal transcripts or an embedded IDE.

## Brand Personality

Quiet, precise, native. Dependable and focused rather than decorative, chatty, or theatrical.

## Anti-references

Do not imitate Codex desktop, Claude Code, or T3 Code visually. Avoid chat-only navigation, embedded-IDE surface sprawl, terminal-painted interfaces, default-full-access affordances, and web chrome pretending to be macOS. Native macOS applications are the interaction-quality reference, especially their windowing, source lists, menus, focus behavior, and platform familiarity.

## Design Principles

1. Make concurrent work and attention needs legible at a glance.
2. Keep the session narrative primary; compress routine machinery without hiding evidence.
3. Preserve Pi semantics and trust boundaries rather than inventing convenient substitutes.
4. Use familiar macOS behavior before custom interaction patterns.
5. Hand editing and Git delivery to the user’s existing tools instead of becoming a miniature IDE.

## Accessibility & Inclusion

Target WCAG 2.2 AA and full keyboard-only operation. Provide logical focus order, visible focus, VoiceOver names and state announcements, text resizing, sufficient contrast, status conveyed by text and icon rather than color alone, and reduced-motion behavior. Prefer standard macOS controls and test with macOS Keyboard Navigation enabled.
