# Product

## Register

product

## Users

Existing Pi users managing coding work across one or more projects. They understand coding agents and Pi’s provider and customization model, and want to orchestrate work without remaining in a terminal interface.

## Product Purpose

PiLot lets users start, monitor, steer, review, and revisit concurrent Pi tasks from a native-feeling desktop interface. Success means adding a graphical task workflow without breaking continuity with the user’s existing Pi environment or replacing their preferred editor and terminal.

## Brand Personality

Focused, capable, transparent. PiLot should feel calm and efficient while making agent actions, task state, cost, and risk legible.

## Anti-references

PiLot must not resemble a terminal emulator, generic chatbot, or VS Code clone. It should not reproduce IDE chrome or hide consequential agent activity behind decorative simplification.

## Design Principles

1. Center tasks, not chat transcripts.
2. Expose state when it changes decisions.
3. Preserve continuity with the Pi ecosystem.
4. Integrate with existing developer tools instead of rebuilding them.
5. Follow host-platform behavior where users expect it.

## Accessibility & Inclusion

Meet WCAG 2.2 AA. Support complete keyboard operation, visible focus, screen-reader semantics, compliant contrast, scalable text, non-color status cues, and reduced-motion preferences.
